# PRD：個人信用卡管理網頁 (creditcard-personal)

## 專案背景

這是一個個人用途的信用卡管理工具，用來解決多卡用戶的兩個核心痛點：
1. 站在付款當下不知道刷哪張卡回饋最高
2. 容易忘記繳費截止日而產生違約金

這不是商業產品，是 Vibe Coding 練習，優先讓功能跑起來，之後再迭代。

---

## 技術規格

- **形式**：單一 `index.html` 檔案，不需要後端、不需要資料庫
- **語言**：HTML + vanilla JavaScript（不用框架）
- **資料儲存**：`localStorage`（存在瀏覽器本地，重開不會消失）
- **介面語言**：繁體中文
- **裝置支援**：手機瀏覽器優先（RWD，可在桌機上也能正常顯示）

## MVP 功能範圍（第一版，只做這三件事）

### 功能一：帳單繳費看板

- 顯示每一張卡的名稱、帳單截止日、繳費截止日
- 自動計算「距離繳費截止日還有幾天」
- 視覺警示規則：
  - 7 天以內：紅色標示
  - 8–14 天：橘色標示
  - 15 天以上：正常顯示

### 功能二：選卡建議

- 從下拉選單選擇消費地點（選項來自上方常消費地點清單）
- 按下查詢後，顯示建議刷哪張卡
- 回饋邏輯用簡單規則寫死（hardcode），不需要複雜演算法
- 初始回饋規則建議（可之後手動修改）：

| 消費地點 | 建議刷卡 | 原因 |
|----------|----------|------|
| 全聯 | CUBE 卡 | 行動支付回饋較高 |
| 7-11 / 全家 | CUBE 卡 | 行動支付加成 |
| 50嵐 / 路易莎 | CUBE 卡 | 行動支付加成 |
| 星巴克 | CUBE 卡 | 行動支付加成 |
| 屈臣氏 | Unicard | 指定通路回饋 |
| 台灣大哥大 | Richart 卡 | 電信帳單回饋 |
| AI訂閱 | CUBE 卡 | 海外交易/訂閱加成 |
| 高鐵 / 國光客運 | 街口聯名卡 | 街口支付加成 |
| 健身工廠 | Richart 卡 | 一般消費回饋穩定 |

### 功能三：本地資料儲存

- 所有資料（卡片資訊、回饋規則）存在 `localStorage`
- 關閉瀏覽器後重開，資料不會消失
- 不需要登入，不需要帳號
